# NatSpec Patterns — Multi-language Reference

NatSpec-style documentation patterns adapted per language.

---

## Universal Tags

| Tag | Required on | Description |
|---|---|---|
| `@notice` | All public/exported declarations | What it does — from the caller's perspective |
| `@dev` | Any declaration | Technical detail for developers |
| `@param` | Every function with parameters | One per param: `@param name Description` |
| `@returns` / `@return` | Every function with a return value | What is returned and what it means |
| `@inheritdoc` | Overrides with no added logic | `@inheritdoc InterfaceName` — replaces all other tags |
| `@custom:...` | Custom metadata | `@custom:security-contact security@example.com` |

### Requirement Rules

```
PUBLIC/EXPORTED → @notice required + @param + @returns (if they exist)
PRIVATE/INTERNAL → @dev if logic is non-obvious (optional)
OVERRIDE WITH NO LOGIC → @inheritdoc (replaces everything else)
OVERRIDE WITH LOGIC → @notice + @dev explaining what's different
```

## TypeScript / JavaScript

Uses JSDoc syntax (`/** ... */`). NatSpec tags are valid inside JSDoc.

### Exported function
```typescript
/**
 * @notice Registers a new user account with email verification.
 * @dev Hashes the password with bcrypt before saving. Sends a verification email on success.
 * @param data - Validated registration form data.
 * @returns Promise resolving to the created user with generated ID, or throws on conflict.
 */
export async function createUser(data: CreateUserRequest): Promise<User> { ... }
```

### React hook
```typescript
/**
 * @notice Hook for creating a new user via API.
 * @dev Invalidates the users list cache on success.
 * @returns React Query mutation object with isPending, mutate, and error.
 */
export function useCreateUser() { ... }
```

### Interface / Type
```typescript
/**
 * @notice Represents a registered user in the system.
 * @dev Maps directly to the users table in the database.
 */
export interface User {
  /** @dev UUID v4 generated by the database */
  id: string;
  /** @notice Display name shown in the UI */
  name: string;
  /** @notice Primary contact email, must be unique */
  email: string;
}
```

### Class method
```typescript
class UserService {
  /**
   * @notice Retrieves a user by their unique ID.
   * @param userId - UUID v4 of the user account.
   * @returns User instance, or null if the ID does not exist.
   */
  async findById(userId: string): Promise<User | null> { ... }

  /**
   * @dev Normalizes email to lowercase and trims whitespace.
   */
  private normalizeEmail(email: string): string { ... }
}
```

## Solidity

Uses `///` (single-line NatSpec) or `/** */` (multi-line). Prefer `///`.

### External/public function
```solidity
/// @notice Deposits tokens into the vault on behalf of the caller.
/// @dev Requires prior approval via ERC20.approve. Emits the Deposited event.
/// @param amount The token amount to deposit in wei.
/// @param token Address of the ERC20 contract to deposit.
/// @return shares The number of vault shares issued to the depositor.
function deposit(uint256 amount, address token) external returns (uint256 shares) { ... }
```

### Event
```solidity
/// @notice Emitted when a deposit is successfully completed.
/// @param user Address of the depositor.
/// @param amount Amount deposited in wei.
/// @param shares Shares issued in exchange.
event Deposited(address indexed user, uint256 amount, uint256 shares);
```

### Custom error
```solidity
/// @notice Thrown when the deposit amount is zero.
error ZeroAmount();

/// @notice Thrown when the caller's balance is insufficient.
/// @param available The caller's current balance.
/// @param required The balance required for the operation.
error InsufficientBalance(uint256 available, uint256 required);
```

### Public state variable
```solidity
/// @notice Address of the contract administrator.
address public owner;
```

### Override with @inheritdoc
```solidity
/// @inheritdoc IVault
function withdraw(uint256 shares) external override returns (uint256 amount) { ... }
```

## Python

Uses docstrings (`"""..."""`). NatSpec tags inside the docstring.

```python
def create_user(data: CreateUserRequest) -> User:
    """
    @notice Registers a new user account with email verification.

    @dev Validates required fields, hashes the password, and sends a
    verification email on success. Raises ValueError for duplicate emails.

    @param data: Validated registration data.
    @returns: Created User instance with generated ID.

    @raises ValueError: If the email already exists in the database.
    @raises DatabaseError: If the write operation fails.
    """
    ...
```

## Rust

Uses `///` (doc comments rendered by `rustdoc`).

```rust
/// @notice Creates a new user account and persists it to the database.
///
/// @dev Validates required fields per the domain rules. Returns Err if the
/// email already exists or if the database write fails.
///
/// # Arguments
/// * `data` - Validated registration request.
///
/// # Returns
/// * `Ok(User)` - Created user with generated ID.
/// * `Err(AppError)` - Validation or database error.
pub async fn create_user(data: CreateUserRequest) -> Result<User, AppError> { ... }
```

## Go

Uses line comments (`//`) before the declaration. GoDoc convention.

## General Rules

### Always document
- All public/exported functions, methods, and constructors with logic
- All exported interfaces, structs, types, and enums
- All events and custom errors (Solidity)
- Public state variables (Solidity)

### Document when logic is non-obvious
- Private/internal functions with complex logic (`@dev` only)
- Constants with non-evident meaning

### Never document
- Trivial getters/setters
- Functions whose name already explains everything (`getById()`, `setName()`)
- Simple constructors with no logic

### Quality standards

```
❌ Bad:  @notice Creates a user.
✅ Good: @notice Registers a new user account with email uniqueness validation
         and password hashing. Sends a verification email on success.

❌ Bad:  @param id The id.
✅ Good: @param id UUID v4 of the target user. Must exist in the database.

❌ Bad:  @returns The user.
✅ Good: @returns User instance with generated ID and creation timestamp,
         or null if the ID was not found.
```
